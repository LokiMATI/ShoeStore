# Как запустить проект
Для начала, необходимо в СУБД MS SQL загрузить последний Backup базы данных.

После этого можно запускать различные проекты и проверять их работоспособность.

## Что не реализовано
- Автоинкрементация новых значений в БД (SQL Server Management Studio 22 во всём виноват. Поломаная программа, с урезанным функционалом и с постоянными вылетами)
- Сохранение изображений локально на компьютере (Файлы сохраняются повторно в БД в качестве varbinary)
- Рефакторинг кода
- Хэширование пароля с помощью BCrtypt
